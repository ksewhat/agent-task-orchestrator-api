# AI Agent Task Orchestrator API 프로그램 정의서

## 1. 프로젝트 개요

**AI Agent Task Orchestrator API**는 사용자의 자연어 요청을 AI Agent가 이해하고, 개인 컨텍스트와 최근 작업 메모리를 참고하여 실행 계획을 수립한 뒤, 필요한 도구를 선택적으로 활용하여 결과물을 생성하는 백엔드 기반 AI Agent 시스템이다.

이 프로젝트는 단순히 LLM API를 호출해 답변을 반환하는 서비스가 아니라, 사용자의 상황을 기억하고, 필요한 정보를 검색하며, 작업 상태를 추적하고, 외부 시스템에 영향을 주는 작업은 사용자 승인 후 실행하는 **실무형 AI Agent Orchestration 시스템**을 목표로 한다.

현재 프로젝트는 FastAPI 기반으로 기본 서버 구조, LLM 연동, Agent 계획 생성, 비동기 작업 처리, 작업 상태 조회, 실행 히스토리 저장 기능까지 구현되어 있다. 이후 확장 방향은 개인 컨텍스트 기반 RAG, 메모리 큐, Tool Calling, 보고서 생성, Google Sheets 연동, 작업 승인 구조를 포함하는 AI Agent Engineering 포트폴리오 프로젝트로 발전시키는 것이다.

---

## 2. 프로젝트 목적

이 프로젝트의 목적은 AI Agent가 단순히 답변을 생성하는 수준을 넘어, 실제 서비스 백엔드 안에서 다음 역할을 수행할 수 있음을 보여주는 것이다.

첫째, 사용자의 자연어 요청을 분석하여 구조화된 실행 계획으로 변환한다.

둘째, 장시간 걸릴 수 있는 LLM 작업을 비동기 방식으로 처리하고, 사용자가 작업 상태와 결과를 추적할 수 있게 한다.

셋째, 사용자의 이력, 프로젝트 상태, 학습 목표, 과거 작업 기록을 개인 컨텍스트로 활용하여 개인화된 결과를 생성한다.

넷째, 최근 요청과 실행 결과를 메모리 큐로 관리하여 Agent가 작업 흐름을 이어서 판단할 수 있게 한다.

다섯째, 필요한 경우 외부 정보 검색, 보고서 생성, Google Sheets 업데이트 등 외부 도구를 활용한다.

여섯째, 외부 시스템을 변경하는 작업은 즉시 실행하지 않고 사용자 승인 후 수행하여 안전한 Agent 실행 구조를 만든다.

---

## 3. 핵심 컨셉

이 프로젝트의 핵심 컨셉은 **“계획을 세우고, 기억하고, 도구를 사용하며, 안전하게 실행하는 AI Agent 백엔드”**이다.

일반적인 LLM API는 사용자의 질문에 대해 한 번의 답변을 생성하는 데 집중한다. 반면 이 프로젝트는 사용자의 요청을 하나의 작업으로 관리하고, 그 작업이 어떤 상태인지 추적하며, 필요한 컨텍스트와 도구를 활용해 결과를 만들어내는 Agent 실행 구조를 지향한다.

전체 흐름은 다음과 같다.

```text
사용자 요청
→ 개인 컨텍스트 및 최근 메모리 참고
→ Agent 실행 계획 생성
→ 필요한 도구 판단
→ 결과물 생성
→ 위험 작업은 승인 대기
→ 승인 후 외부 작업 실행
→ 실행 결과와 히스토리 저장
```

## 4. 주요 기능 정의

## 4.1 자연어 기반 Agent 계획 생성

사용자가 자연어로 요청을 입력하면 AI Agent가 요청의 목표, 실행 단계, 예상 리스크, 다음 액션을 구조화하여 계획으로 생성한다.

이 기능은 단순 질의응답이 아니라, 사용자의 요청을 실제 실행 가능한 작업 단위로 해석하는 역할을 한다. 이를 통해 사용자는 막연한 요청을 구체적인 실행 계획으로 전환할 수 있다.

**적용 기술스택**

```text
FastAPI
OpenAI API
Pydantic
```

**포트폴리오 의미**

이 기능은 LLM을 단순 챗봇처럼 사용하는 것이 아니라, 백엔드 서비스 안에서 구조화된 결과를 생성하는 방식으로 활용할 수 있음을 보여준다.

---

## 4.2 비동기 Agent 작업 처리

LLM 기반 작업은 응답 시간이 길어질 수 있기 때문에, 사용자의 요청을 즉시 완료 결과로 반환하지 않고 하나의 작업으로 등록한 뒤 백그라운드에서 처리한다.

사용자는 작업이 진행 중인지, 성공했는지, 실패했는지 확인할 수 있으며, 완료된 작업의 결과를 다시 조회할 수 있다.

**적용 기술스택**

```text
FastAPI BackgroundTasks
Redis
RQ
```

**포트폴리오 의미**

이 기능은 시간이 오래 걸리는 AI 작업을 안정적으로 처리하기 위한 백엔드 작업 관리 구조를 보여준다. AI Agent 서비스에서 중요한 비동기 처리, 상태 관리, 결과 추적 역량을 표현할 수 있다.

---

## 4.3 실행 히스토리 관리

Agent가 처리한 요청과 결과 요약을 히스토리로 저장한다. 이를 통해 사용자는 과거에 어떤 요청을 했고, 어떤 결과가 생성되었는지 확인할 수 있다.

히스토리는 단순 로그가 아니라, 이후 Agent가 사용자의 작업 흐름을 이해하는 데 활용될 수 있는 기반 데이터가 된다.

**적용 기술스택**

```text
PostgreSQL
SQLAlchemy
```

**포트폴리오 의미**

이 기능은 AI Agent의 실행 결과를 일회성으로 버리지 않고, 서비스 데이터로 관리하는 구조를 보여준다. 또한 백엔드 관점에서 데이터 영속성, 조회, 이력 관리 역량을 드러낼 수 있다.

---

## 4.4 메모리 큐 기반 최근 작업 흐름 관리

Agent는 모든 정보를 장기 저장하지 않고, 최근 사용자 요청, Agent 응답, 도구 실행 결과, 실패 원인 등을 메모리 큐에 순차적으로 저장한다.

메모리 큐는 Agent가 최근 흐름을 놓치지 않고 이어서 판단할 수 있게 도와준다. 예를 들어 직전 작업이 실패했다면 다음 요청에서 그 실패 원인을 참고할 수 있고, 사용자가 “방금 계획한 내용 기준으로 다시 정리해줘”라고 요청했을 때 최근 작업 내용을 활용할 수 있다.

또한 메모리 큐에 쌓인 정보 중 중요한 내용은 장기 기억이나 검색 가능한 지식 저장소로 승격할 수 있다.

**적용 기술스택**

```text
Redis
RQ
```

**포트폴리오 의미**

이 기능은 Agent의 메모리 구조를 단순 대화 기록이 아니라, 최근 작업 흐름을 관리하는 큐 기반 구조로 설계했다는 점을 보여준다. AI Agent Engineering에서 중요한 memory handling, context management, stateful agent 설계 역량을 표현할 수 있다.

---

## 4.5 개인 컨텍스트 기반 RAG

사용자의 경력, 프로젝트 상태, 학습 목표, 기술스택, 과거 작업 기록 등을 개인 컨텍스트로 저장하고, Agent가 계획을 생성할 때 관련 정보를 검색해 활용한다.

이를 통해 Agent는 일반적인 답변이 아니라 사용자에게 맞는 개인화된 계획과 결과물을 생성할 수 있다.

예를 들어 AI Agent 엔지니어링 포트폴리오를 준비하는 사용자라면, 현재 프로젝트 구현 상태와 보유 기술, 학습 목표를 반영하여 다음 개발 단계를 추천할 수 있다.

**적용 기술스택**

```text
Chroma
OpenAI Embeddings
OpenAI API
```

**포트폴리오 의미**

이 기능은 RAG를 단순 문서 검색이 아니라 개인화된 Agent 계획 생성에 활용하는 구조를 보여준다. 사용자 컨텍스트를 검색하고, LLM 응답에 반영하는 AI Agent 핵심 기술을 표현할 수 있다.

---

## 4.6 외부 정보 검색 기반 계획 보강

Agent가 내부 정보만으로 충분하지 않은 경우 외부 정보를 검색하여 계획에 반영한다.

예를 들어 최신 AI Agent 엔지니어 채용 기술스택, 특정 프레임워크의 최신 사용 흐름, 포트폴리오 트렌드 등을 검색하고, 현재 프로젝트 상태와 비교해 다음 액션을 제안할 수 있다.

**적용 기술스택**

```text
OpenAI Tool Calling
Tavily Search API
```

**포트폴리오 의미**

이 기능은 Agent가 내부 지식에만 의존하지 않고, 필요한 시점에 외부 도구를 활용해 최신 정보를 반영할 수 있음을 보여준다. Tool Calling과 정보 검색을 결합한 Agent 설계 역량을 표현할 수 있다.

---

## 4.7 Tool Calling 기반 도구 실행

Agent는 사용자의 요청을 분석한 뒤 필요한 도구를 선택해 실행한다.

도구는 단순 내부 함수가 아니라 Agent가 사용할 수 있는 기능 단위다. 예를 들어 히스토리 조회, 개인 컨텍스트 검색, 외부 검색, 보고서 생성, Google Sheets 업데이트 등이 도구가 될 수 있다.

이 구조를 통해 Agent는 단순히 텍스트를 생성하는 역할을 넘어 실제 작업을 수행하는 실행 주체로 확장된다.

**적용 기술스택**

```text
OpenAI Tool Calling
Pydantic
```

**포트폴리오 의미**

이 기능은 AI Agent Engineering에서 핵심적인 “도구를 사용하는 Agent” 구조를 보여준다. LLM이 단순히 답변을 생성하는 것이 아니라, 필요한 기능을 판단하고 호출하는 구조를 설계할 수 있음을 나타낸다.

---

## 4.8 보고서 형태 결과물 생성

Agent가 생성한 계획을 단순 JSON 응답이 아니라 사람이 읽기 좋은 보고서 형태로 정리한다.

결과물은 프로젝트 기획안, 학습 계획서, 실행 계획서, 기술 검토서와 같은 문서형 산출물로 활용할 수 있다.

초기 결과물은 Markdown 형식으로 생성하며, 향후 PDF나 문서 파일 형태로 확장할 수 있다.

**적용 기술스택**

```text
OpenAI API
Markdown
```

**포트폴리오 의미**

이 기능은 AI Agent가 단순 API 응답이 아니라 실제 업무에 활용 가능한 산출물을 생성할 수 있음을 보여준다. 기획, 문서화, 보고서 자동화 기능을 포함한 Agent 활용 사례로 설명할 수 있다.

---

## 4.9 Google Sheets 연동

Agent가 생성한 계획이나 결과를 Google Sheets에 정리하여 외부 산출물로 관리할 수 있게 한다.

예를 들어 주차별 학습 계획, 프로젝트 구현 로드맵, 작업 상태표, 포트폴리오 기능 목록 등을 Google Sheets에 자동으로 정리할 수 있다.

**적용 기술스택**

```text
Google Sheets API
OpenAI Tool Calling
```

**포트폴리오 의미**

이 기능은 Agent가 외부 서비스와 연동하여 실제 업무 도구에 결과를 반영할 수 있음을 보여준다. AI Agent가 단순 분석을 넘어 사용자의 작업 환경에 직접 연결되는 구조를 표현할 수 있다.

---

## 4.10 작업 전 승인 구조

Agent가 외부 시스템을 변경하거나 중요한 작업을 수행할 때는 즉시 실행하지 않고 사용자 승인을 기다린다.

예를 들어 Google Sheets 업데이트, 외부 API 쓰기 작업, 데이터 삭제, 이메일 발송과 같은 작업은 사용자에게 실행 내용을 미리 보여주고 승인 후 처리한다.

이 구조는 AI Agent가 자율적으로 도구를 사용하되, 위험한 작업은 사람이 통제할 수 있게 만드는 안전장치다.

**적용 기술스택**

```text
PostgreSQL
Redis
FastAPI
```

**포트폴리오 의미**

이 기능은 AI Agent 서비스에서 중요한 안전성 설계를 보여준다. 단순 자동화가 아니라, 사용자의 승인과 통제를 포함한 Human-in-the-loop 구조를 설계할 수 있음을 표현한다.

## 5. 전체 기술스택

이 프로젝트는 다음 기술스택을 기준으로 구성한다.

| 영역                     | 기술                  |
| ---------------------- | ------------------- |
| Backend Framework      | FastAPI             |
| Data Validation        | Pydantic            |
| Environment Management | pydantic-settings   |
| LLM Integration        | OpenAI API          |
| Tool Calling           | OpenAI Tool Calling |
| Vector Database        | Chroma              |
| Embedding              | OpenAI Embeddings   |
| Memory Queue           | Redis               |
| Async Job Queue        | RQ                  |
| Database               | PostgreSQL          |
| ORM                    | SQLAlchemy          |
| External Search        | Tavily Search API   |
| External Integration   | Google Sheets API   |
| Report Format          | Markdown            |

이 기술스택은 무료로 시작하기 쉽고, AI Agent 백엔드 포트폴리오에서 설명하기 좋은 조합이다. FastAPI는 Python 기반 API 서버 구현에 적합하고, PostgreSQL과 Redis는 백엔드 상태 관리와 큐 처리의 기본 역량을 보여준다. Chroma와 OpenAI Embeddings는 개인 컨텍스트 검색을 위한 RAG 구조를 만들 수 있으며, OpenAI Tool Calling은 Agent가 도구를 선택하고 실행하는 구조를 구현하는 데 사용된다.

---

## 6. 프로그램 차별점

이 프로젝트의 차별점은 LLM을 단순 호출하는 API가 아니라, 실제 Agent 서비스에 필요한 백엔드 구조를 함께 설계한다는 점이다.

첫째, 사용자의 요청을 즉시 답변으로 끝내지 않고 작업 단위로 관리한다.

둘째, Agent가 최근 작업 흐름을 기억할 수 있도록 메모리 큐를 사용한다.

셋째, 개인 컨텍스트를 RAG로 검색하여 사용자 맞춤형 결과를 생성한다.

넷째, 필요한 경우 외부 검색과 외부 도구를 활용한다.

다섯째, 위험한 작업은 사용자 승인 후 실행하는 안전 구조를 포함한다.

여섯째, 실행 결과와 히스토리를 저장하여 Agent의 작업 과정을 추적할 수 있다.

---


## 8. 최종 정의

**AI Agent Task Orchestrator API**는 사용자의 자연어 요청을 기반으로 개인 컨텍스트와 최근 메모리를 참고하고, 필요한 도구를 선택적으로 활용하여 실행 계획과 결과물을 생성하는 FastAPI 기반 AI Agent 백엔드 시스템이다.

이 시스템은 LLM 호출, 비동기 작업 처리, 메모리 큐, RAG, Tool Calling, 외부 서비스 연동, 작업 승인 구조를 결합하여 실무형 AI Agent Engineering 역량을 보여주는 포트폴리오 프로젝트를 목표로 한다.
